# 인수인계 문서 작성 워크플로우

## 📋 개요

이 워크플로우는 작업이 여러 AI 세션에 걸쳐 완료되어야 하거나, 후속 작업이 필요할 때 인수인계 문서를 작성하는 프로세스를 정의합니다.

## 🎯 인수인계 문서 작성 시점

다음 상황에서 인수인계 문서를 작성합니다:

- 작업이 여러 AI 세션에 걸쳐 완료되어야 할 때
- 리팩토링이나 최적화 작업이 여러 파일/모듈에 걸칠 때
- 사용자가 명시적으로 인수인계 문서를 요청할 때
- AI가 판단하기에 후속 작업이 필요할 때
- 복잡한 변경이 여러 부분에 영향을 줄 때

## 🔄 워크플로우 단계

### 1단계: 인수인계 필요성 판단 (Handoff Decision)

**목표**: 인수인계 문서가 필요한지 판단합니다.

**판단 기준**:

- [ ] 작업이 완료되지 않았고 후속 작업이 필요한가?
- [ ] 여러 파일/모듈에 걸친 변경인가?
- [ ] 사용자가 명시적으로 요청했는가?
- [ ] 다음 AI 세션에서 이어서 작업해야 하는가?

**결정**: 위 조건 중 하나라도 해당하면 인수인계 문서 작성

---

### 2단계: 문서 작성 (Document Creation)

**목표**: 인수인계 문서를 작성합니다.

**작업 내용**:

- [ ] 파일명 생성
  - 형식: `[인덱스]_[작업 간략 설명]_[날짜].md`
  - 예시: `01_vnode-normalization-refactor_20250115.md`
  - 위치: `.cursor/mockdowns/` 디렉토리
- [ ] 문서 내용 작성
  - 작업 개요 및 목적
  - 완료된 작업 목록
  - 진행 중인 작업
  - 다음 단계 작업 (우선순위 포함)
  - 관련 파일 목록
  - 주의사항 및 고려사항
  - 참고 자료

**출력물**:

- 인수인계 문서 파일
- 문서 내용

---

### 3단계: 문서 구조 작성 (Document Structure)

**필수 섹션**:

```markdown
# [작업 제목]

## 📋 작업 개요

- 작업 목적과 배경 설명
- 변경된 주요 파일 목록

## ✅ 완료된 작업

- 구현/수정된 기능 목록
- 변경 사항 상세 설명

## 🔄 진행 중인 작업

- 현재 진행 중인 부분
- 예상 완료 시점

## 📝 다음 단계 작업

- 남은 작업 항목
- 우선순위가 높은 작업
- 주의사항 및 고려사항

## 🎯 관련 파일

- 수정된 파일 경로 목록
- 참고해야 할 파일 목록

## ⚠️ 주의사항

- 특별히 주의해야 할 사항
- 테스트 시 확인해야 할 부분
- 잠재적 이슈나 리스크

## 📚 참고 자료

- 관련 문서나 참고 링크
```

---

### 4단계: 문서 품질 확인 (Quality Check)

**확인 사항**:

- [ ] 파일명이 명명 규칙을 따르는가?
- [ ] 모든 사용자용 내용이 한글로 작성되었는가?
- [ ] 파일 경로가 정확하고 완전한가?
- [ ] 다음 단계가 명확히 정의되었는가?
- [ ] 주의사항과 고려사항이 문서화되었는가?
- [ ] 코드 예제가 포함되었는가? (도움이 되는 경우)

---

## 📝 문서 작성 가이드라인

### 파일 명명 규칙

- **형식**: `[인덱스]_[작업 간략 설명]_[날짜].md`
- **인덱스**: 순차적 번호 (01, 02, 03, ...)
- **작업 설명**: 한글, 소문자, 하이픈 사용
- **날짜**: YYYYMMDD 형식

**예시**:

- `01_vnode-정규화-리팩토링_20250115.md`
- `02_reconciliation-최적화_20250116.md`
- `03_hooks-상태관리-개선_20250117.md`

### 내용 작성 규칙

1. **한글 사용**: 사용자에게 보여지는 모든 내용은 한글로 작성
2. **이모지 사용**: 가독성을 위해 적절한 이모지 사용
   - 📋 작업 개요
   - ✅ 완료된 작업
   - 🔄 진행 중인 작업
   - 📝 다음 단계
   - 🎯 관련 파일
   - ⚠️ 주의사항
   - 📚 참고 자료
3. **구체성**: 파일 경로와 함수명을 구체적으로 명시
4. **코드 예제**: 도움이 되는 경우 코드 예제 포함
5. **의존성 명시**: 후속 작업을 위한 의존성이나 전제 조건 명시

## 🔄 문서 업데이트

### 업데이트 시점

- 관련 작업이 계속될 때
- 완료된 섹션을 명확히 표시할 때
- 새로운 정보가 추가될 때

### 업데이트 방법

- 완료된 섹션에 ✅ 표시
- 진행 상황 업데이트
- 다음 단계 우선순위 조정
- 새로운 주의사항 추가

---

## 📚 문서 관리

### 인덱스 관리

- 순차적 번호 유지
- 중복 번호 방지
- 최신 문서 확인

### 아카이브

- 오래된 문서는 아카이브하거나 제거
- 완료된 작업 문서는 보관
- 참고용 문서는 유지

## 💡 예시

### 완성된 인수인계 문서 예시

```markdown
# VNode 정규화 리팩토링

## 📋 작업 개요

- `normalizeNode` 함수의 타입 처리 개선
- null/undefined/boolean 값 필터링 로직 개선
- 변경 파일: `packages/react/src/core/elements.ts`

## ✅ 완료된 작업

- `normalizeNode` 함수의 기본 로직 구현
- null/undefined/boolean 필터링 추가
- 배열 평탄화 로직 추가

## 🔄 진행 중인 작업

- 중첩 배열 처리 최적화
- 타입 가드 개선

## 📝 다음 단계 작업

1. **우선순위 높음**: 중첩 배열의 깊은 평탄화 처리
2. **우선순위 중간**: 타입 가드 함수 개선
3. **우선순위 낮음**: 성능 최적화

## 🎯 관련 파일

- `packages/react/src/core/elements.ts` (수정됨)
- `packages/react/src/utils/validators.ts` (참고)
- `packages/react/src/__tests__/basic.mini-react.test.tsx` (테스트)

## ⚠️ 주의사항

- 테스트 파일은 수정하지 않음
- `isEmptyValue` 함수의 동작 확인 필요
- 깊은 중첩 배열의 경우 성능 고려 필요

## 📚 참고 자료

- `.cursor/rules/problem-solving-workflow.mdc`
- `docs/01-implementation-guide.md`
```
